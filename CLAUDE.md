# CLAUDE.md — genieacs-ha

## Overview
Home Assistant custom integration for managing TR-069 CPE devices (routers, ONTs, gateways) through a GenieACS instance. Connects to the GenieACS NBI REST API (port 7557) to surface device status, WAN IP, uptime, and firmware info.

## Tech Stack
- Python (Home Assistant custom component)
- HACS-compatible integration
- pytest for testing
- Codecov for coverage
- GitHub Actions CI

## Development
```bash
pip install -r requirements-test.txt
pytest tests/
```

Install via HACS or copy `custom_components/genieacs/` into HA config directory.

## Architecture
- `custom_components/genieacs/` — HA integration code (config flow, sensors, binary sensors)
- `tests/` — test suite
- `hacs.json` — HACS metadata
- `docs/` — documentation and images

## Key Rules
- Requires a running GenieACS instance with NBI accessible (default port 7557)
- Minimum Home Assistant version: 2024.1
- Device is considered online if it reported within the last 5 minutes
- Never hardcode ACS URLs or credentials; configured via HA config flow
- Part of the broader GenieACS ecosystem (container, MCP, ansible, sim)
